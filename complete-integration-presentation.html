1단계: Claude Code 설정

1

Claude Code OAuth 토큰 발급

Claude Code 설정에서 OAuth 토큰을 생성합니다.

# Claude Code 웹 인터페이스에서 Settings → API Keys → Generate OAuth Token # 생성된 토큰을 안전한 곳에 저장
2

로컬 Claude 설정 파일 생성

📁 .claude/settings.local.json
{ "permissions": { "allow": [ "mcp__mcp-atlassian__jira_add_comment", "Bash(git checkout:*)", "Bash(git add:*)", "Bash(git commit:*)", "Bash(git push:*)", "Bash(gh pr create:*)", "Bash(gh pr list:*)" ], "deny": [], "ask": [] }, "enableAllProjectMcpServers": true, "enabledMcpjsonServers": ["mcp-atlassian"] }
3

권한 설정 확인

Claude Code가 필요한 도구들에 접근할 수 있도록 권한을 설정합니다.

  • Git 명령어: 브랜치 생성, 커밋, 푸시 권한
  • GitHub CLI: PR 생성, 관리 권한
  • Jira MCP: 티켓 업데이트 권한

✅ 설정 완료 확인

Claude Code에서 mcp 명령어로 MCP 서버 상태를 확인할 수 있습니다.

2단계: Jira MCP 서버 설정

1

Jira API 토큰 생성

🔐 Jira API Token 생성 과정
  1. Atlassian 계정 설정 접속 (id.atlassian.com)
  2. 보안 탭으로 이동
  3. API 토큰 생성 클릭
  4. 라벨: "Claude-GitHub Integration" 입력
  5. 만료 기간 설정 (권장: 1년)
  6. 생성된 토큰 안전한 곳에 저장
2

MCP 서버 설정 파일 생성

📁 .mcp.json (프로젝트 루트)
{ "mcpServers": { "mcp-atlassian": { "command": "uvx", "args": [ "mcp-atlassian", "--jira-url=https://your-domain.atlassian.net", "--jira-username=your-email@company.com", "--jira-token=YOUR_API_TOKEN_HERE" ] } } }
⚠️ 보안 주의: 실제 토큰 값은 환경변수나 별도 보안 저장소 사용 권장
3

연결 테스트

# Claude Code에서 테스트 /mcp list # mcp-atlassian 서버가 활성화되었는지 확인 # Jira 연동 테스트 Jira에서 CX-TEST 티켓을 검색해보세요

3단계: GitHub Actions 워크플로우

1

GitHub Secrets 설정

🔐 Repository → Settings → Secrets
Secret Name Value 설명
CLAUDE_CODE_OAUTH_TOKEN 1단계에서 생성한 토큰 Claude API 인증
JIRA_API_TOKEN 2단계에서 생성한 토큰 Jira API 인증
2

워크플로우 파일 생성

📁 .github/workflows/claude-code-review.yml
name: Claude Code Review & Jira Integration on: pull_request: types: [opened, synchronize] jobs: automated-review: runs-on: ubuntu-latest permissions: contents: read pull-requests: write issues: read id-token: write steps: # 1. 코드 체크아웃 - name: Checkout repository uses: actions/checkout@v4 with: fetch-depth: 1 # 2. Jira 티켓 번호 추출 - name: Extract Jira Ticket id: extract_jira run: | # PR 브랜치명에서 티켓 번호 추출 BRANCH_NAME="${{ github.head_ref }}" TICKET=$(echo "$BRANCH_NAME" | grep -o 'CX-[0-9]\+' | head -n1) echo "ticket=$TICKET" >> $GITHUB_OUTPUT echo "Found ticket: $TICKET" # 3. Claude 코드 리뷰 실행 - name: Run Claude Code Review uses: anthropics/claude-code-action@v1 with: claude_code_oauth_token: ${{ secrets.CLAUDE_CODE_OAUTH_TOKEN }} prompt: | 이 PR을 리뷰하고 다음 사항들을 검토해주세요: - 코드 품질 및 베스트 프랙티스 - 잠재적 버그나 이슈 - 성능 고려사항 - 보안 취약점 - 테스트 커버리지 리뷰 결과를 gh pr comment 명령어로 PR에 코멘트로 남겨주세요. # 4. Jira 티켓 자동 업데이트 - name: Update Jira Ticket if: steps.extract_jira.outputs.ticket != '' uses: anthropics/claude-code-action@v1 with: claude_code_oauth_token: ${{ secrets.CLAUDE_CODE_OAUTH_TOKEN }} prompt: | Jira 티켓 ${{ steps.extract_jira.outputs.ticket }}에 다음 정보로 코멘트를 추가해주세요: **🤖 GitHub Actions 자동 업데이트** - PR: ${{ github.event.pull_request.html_url }} - 브랜치: ${{ github.head_ref }} - 작성자: ${{ github.event.pull_request.user.login }} - 상태: 코드 리뷰 진행 중 Claude 코드 리뷰가 완료되었습니다.
3

워크플로우 권한 설정

Actions가 필요한 권한을 정확히 설정해야 합니다:

  • contents: read - 코드 읽기 권한
  • pull-requests: write - PR 코멘트 작성
  • issues: read - 이슈 정보 조회
  • id-token: write - OIDC 토큰 발급

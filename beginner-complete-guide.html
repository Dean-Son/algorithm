    방법 1: 환경변수 사용 (보안 권장)

    📸 터미널 스크린샷: 환경변수 설정 과정

    • 화면: VSCode 터미널 창 또는 iTerm/Terminal

    • 명령어 실행 후: ".env 파일 생성됨" 메시지

    • 파일 탐색기: .env, .gitignore 파일 표시

    📸 파일 구조 스크린샷: 생성된 파일들

    • 프로젝트 폴더에 .env, .gitignore, .mcp.json 표시

    • VSCode 좌측 파일 탐색기에서 확인 가능

    • .env 파일에 🔒 아이콘으로 보안 파일 표시

    # 1. 환경변수 설정 파일 생성 cat > .env << 'EOF' JIRA_URL=https://your-company.atlassian.net JIRA_EMAIL=your-email@company.com JIRA_TOKEN=your-api-token-here EOF # 2. .env 파일을 git에서 제외 echo ".env" >> .gitignore # 3. .mcp.json 파일 생성 (환경변수 사용) cat > .mcp.json << 'EOF' { "mcpServers": { "mcp-atlassian": { "command": "uvx", "args": [ "mcp-atlassian", "--jira-url=${JIRA_URL}", "--jira-username=${JIRA_EMAIL}", "--jira-token=${JIRA_TOKEN}" ] } } } EOF
    ⚠️
    방법 2: 직접 입력 (테스트용만 권장)
    # .mcp.json 파일 직접 생성 (실제 값 입력 필요) cat > .mcp.json << 'EOF' { "mcpServers": { "mcp-atlassian": { "command": "uvx", "args": [ "mcp-atlassian", "--jira-url=https://your-company.atlassian.net", "--jira-username=your-email@company.com", "--jira-token=ATATT3xFfGF0DpY7WXI5aZnO..." ] } } } EOF # 중요: 이 방식 사용 시 .mcp.json을 git에 커밋하지 마세요! echo ".mcp.json" >> .gitignore

    # GitHub Actions 폴더 생성 mkdir -p .github/workflows # 워크플로우 파일 생성 cat > .github/workflows/claude-code-review.yml << 'EOF' name: Claude Code Review & Jira Integration on: pull_request: types: [opened, synchronize, reopened] # 특정 파일만 감지하고 싶다면 아래 주석 해제 # paths: # - "src/**/*.js" # - "src/**/*.ts" # - "src/**/*.py" jobs: automated-review: runs-on: ubuntu-latest permissions: contents: read pull-requests: write issues: read id-token: write steps: # 1. 저장소 체크아웃 - name: Checkout repository uses: actions/checkout@v4 with: fetch-depth: 2 # 변경사항 비교를 위해 2개 커밋 가져오기 # 2. Jira 티켓 번호 추출 - name: Extract Jira ticket from branch name id: extract_jira run: | BRANCH_NAME="${{ github.head_ref }}" echo "Branch name: $BRANCH_NAME" # CX-1234 형태의 티켓 번호 추출 TICKET=$(echo "$BRANCH_NAME" | grep -o 'CX-[0-9]\+' | head -n1) if [ -z "$TICKET" ]; then echo "No Jira ticket found in branch name: $BRANCH_NAME" echo "ticket=" >> $GITHUB_OUTPUT else echo "Found ticket: $TICKET" echo "ticket=$TICKET" >> $GITHUB_OUTPUT fi # 3. 변경된 파일 목록 생성 - name: Get changed files id: changed_files run: | echo "Getting changed files..." CHANGED_FILES=$(git diff --name-only HEAD~1 HEAD || echo "") echo "Changed files: $CHANGED_FILES" echo "files<> $GITHUB_OUTPUT echo "$CHANGED_FILES" >> $GITHUB_OUTPUT echo "EOF" >> $GITHUB_OUTPUT # 4. Claude 코드 리뷰 실행 - name: Run Claude Code Review if: steps.changed_files.outputs.files != '' uses: anthropics/claude-code-action@v1 with: claude_code_oauth_token: ${{ secrets.CLAUDE_CODE_OAUTH_TOKEN }} prompt: | 이 PR을 체계적으로 리뷰해주세요. **분석해야 할 변경 파일들:** ${{ steps.changed_files.outputs.files }} **리뷰 체크리스트:** 1. 🐛 버그 및 로직 오류 2. 🔒 보안 취약점 3. ⚡ 성능 이슈 4. 📏 코드 품질 및 스타일 5. 🧪 테스트 커버리지 6. 📚 문서화 필요성 **리뷰 결과를 GitHub PR에 코멘트로 남겨주세요.** - 발견된 이슈가 있다면 구체적인 수정 방안 제시 - 코드 개선 제안사항 포함 - 긍정적인 부분도 언급 gh pr comment 명령어를 사용해 코멘트를 작성해주세요. # 5. Jira 티켓 자동 업데이트 - name: Update Jira ticket if: steps.extract_jira.outputs.ticket != '' uses: anthropics/claude-code-action@v1 with: claude_code_oauth_token: ${{ secrets.CLAUDE_CODE_OAUTH_TOKEN }} prompt: | Jira 티켓 ${{ steps.extract_jira.outputs.ticket }}에 PR 진행상황을 업데이트해주세요. **업데이트할 정보:** - PR URL: ${{ github.event.pull_request.html_url }} - 브랜치: ${{ github.head_ref }} - 작성자: ${{ github.event.pull_request.user.login }} - 변경 파일 수: $(echo "${{ steps.changed_files.outputs.files }}" | wc -l) - 상태: 🤖 AI 코드 리뷰 완료 - 일시: $(date '+%Y-%m-%d %H:%M:%S') **코멘트 형식:** ``` 🤖 **GitHub Actions 자동 업데이트** **PR 정보:** - 링크: ${{ github.event.pull_request.html_url }} - 브랜치: ${{ github.head_ref }} - 작성자: ${{ github.event.pull_request.user.login }} - 제목: ${{ github.event.pull_request.title }} **상태:** ✅ AI 코드 리뷰 완료 **처리일시:** $(date '+%Y-%m-%d %H:%M:%S KST') ``` Jira MCP를 사용해서 이 정보로 코멘트를 추가해주세요. # 6. 리뷰 완료 알림 - name: Review completion summary if: always() run: | echo "## 📋 리뷰 완료 요약" >> $GITHUB_STEP_SUMMARY echo "- **브랜치:** ${{ github.head_ref }}" >> $GITHUB_STEP_SUMMARY echo "- **Jira 티켓:** ${{ steps.extract_jira.outputs.ticket || 'N/A' }}" >> $GITHUB_STEP_SUMMARY echo "- **변경 파일:** $(echo "${{ steps.changed_files.outputs.files }}" | wc -l)개" >> $GITHUB_STEP_SUMMARY echo "- **상태:** ✅ 자동 리뷰 완료" >> $GITHUB_STEP_SUMMARY EOF # 파일 생성 확인 ls -la .github/workflows/

    # RULES.md 파일 생성 cat > RULES.md << 'EOF' # 개발 규칙 (Development Rules) ## 브랜치 네이밍 규칙 ### Feature 브랜치 - **형식**: `feature/CX-{숫자}` - **규칙**: CX-{숫자}는 반드시 Jira 티켓 번호와 동일해야 합니다 - **예시**: - `feature/CX-1790` (Jira 티켓: CX-1790) - `feature/CX-2024` (Jira 티켓: CX-2024) ### 기타 브랜치 유형 - **Hotfix**: `hotfix/CX-{숫자}` - **Bugfix**: `bugfix/CX-{숫자}` - **Release**: `release/v{버전}` ## 자동화 시스템 사용법 ### PR 생성 시 주의사항 1. 브랜치명에 정확한 Jira 티켓 번호 포함 2. PR 제목에 티켓 번호 명시: `[CX-1234] 기능 설명` 3. PR 설명에 관련 Jira 티켓 링크 포함 ### AI 코드 리뷰 활용 - PR 생성 후 2-3분 내에 자동 리뷰 댓글 확인 - Claude가 제안한 개선사항 적극 반영 - 보안/성능 이슈 발견 시 즉시 수정 ## 주의사항 - 브랜치명 규칙 준수 필수 (자동화 작동 조건) - API 토큰은 정기적으로 갱신 (3개월 주기 권장) - 민감한 정보는 환경변수 사용 EOF # 팀과 공유 git add RULES.md git commit -m "docs: Add development rules and automation guidelines" git push origin main

    # GitHub Actions 성능 모니터링 추가 cat >> .github/workflows/claude-code-review.yml << 'EOF' # 성능 메트릭 수집 - name: Performance Metrics if: always() run: | echo "## 📊 성능 메트릭" >> $GITHUB_STEP_SUMMARY echo "- **워크플로우 실행 시간:** $(date -d "${{ github.event.pull_request.created_at }}" '+%s') 초" >> $GITHUB_STEP_SUMMARY echo "- **변경 파일 수:** $(echo "${{ steps.changed_files.outputs.files }}" | wc -l)" >> $GITHUB_STEP_SUMMARY echo "- **브랜치:** ${{ github.head_ref }}" >> $GITHUB_STEP_SUMMARY echo "- **실행 환경:** ubuntu-latest" >> $GITHUB_STEP_SUMMARY # 비용 추적 - name: Cost Tracking if: always() run: | echo "## 💰 비용 정보" >> $GITHUB_STEP_SUMMARY echo "- **Actions 사용 시간:** 약 2-3분" >> $GITHUB_STEP_SUMMARY echo "- **Claude API 호출:** 1-2회" >> $GITHUB_STEP_SUMMARY echo "- **Jira API 호출:** 1회" >> $GITHUB_STEP_SUMMARY EOF

    팀 온보딩 자동화 스크립트

    # setup-automation.sh 스크립트 생성 cat > setup-automation.sh << 'EOF' #!/bin/bash # Claude-GitHub-Jira 자동화 설정 스크립트 echo "🚀 Claude-GitHub-Jira 자동화 설정을 시작합니다..." # 필수 도구 설치 확인 echo "📋 필수 도구 설치 상태 확인..." command -v node >/dev/null 2>&1 || { echo "Node.js가 필요합니다. 설치해주세요."; exit 1; } command -v git >/dev/null 2>&1 || { echo "Git이 필요합니다. 설치해주세요."; exit 1; } command -v gh >/dev/null 2>&1 || { echo "GitHub CLI가 필요합니다. 설치해주세요."; exit 1; } command -v uvx >/dev/null 2>&1 || { echo "uvx가 필요합니다. pip install uv로 설치해주세요."; exit 1; } echo "✅ 모든 필수 도구가 설치되어 있습니다." # 디렉토리 생성 echo "📁 디렉토리 구조 생성 중..." mkdir -p .claude mkdir -p .github/workflows # 설정 파일 복사 (템플릿 기반) echo "⚙️ 설정 파일 생성 중..." echo "환경변수 파일 .env를 직접 편집하여 본인의 토큰 정보를 입력해주세요." # 완료 메시지 echo "🎉 기본 설정 완료!" echo "📖 다음 단계:" echo " 1. .env 파일에 실제 토큰 값 입력" echo " 2. GitHub Repository Secrets 설정" echo " 3. 테스트 PR 생성하여 동작 확인" echo "" echo "❓ 문제가 발생하면 RULES.md 파일을 참조하세요." EOF chmod +x setup-automation.sh

    📖 팀원 온보딩 가이드

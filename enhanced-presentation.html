📁 RULES.md
# 개발 규칙 (Development Rules) ## 브랜치 네이밍 규칙 (Branch Naming Convention) ### Feature 브랜치 - **형식**: `feature/CX-{숫자}` - **규칙**: CX-{숫자}는 반드시 Jira 티켓 번호와 동일해야 합니다 - **예시**: - `feature/CX-1790` (Jira 티켓: CX-1790) - `feature/CX-2024` (Jira 티켓: CX-2024) ### 기타 브랜치 유형 - **Hotfix**: `hotfix/CX-{숫자}` - **Bugfix**: `bugfix/CX-{숫자}` - **Release**: `release/v{버전}`

🎯 실제 적용 사례

# 실제 브랜치 생성 명령어 git checkout main git checkout -b feature/CX-1790 # 자동화 스크립트에서 티켓 번호 추출 TICKET=$(echo "feature/CX-1790" | grep -o 'CX-[0-9]\+') echo "티켓 번호: $TICKET" # 출력: 티켓 번호: CX-1790
  • 100% 자동 매칭: 브랜치명에서 Jira 티켓 번호 자동 추출

name: Claude Code Review on: pull_request: types: [opened, synchronize] jobs: claude-review: runs-on: ubuntu-latest steps: - name: Checkout repository uses: actions/checkout@v4 - name: Run Claude Code Review uses: anthropics/claude-code-action@v1 with: claude_code_oauth_token: ${{ secrets.CLAUDE_CODE_OAUTH_TOKEN }} prompt: | Please review this pull request and provide feedback on: - Code quality and best practices - Potential bugs or issues - Performance considerations - Security concerns - name: Extract Jira Ticket run: | TICKET=$(echo "${{ github.event.pull_request.body }}" | grep -o 'CX-[0-9]\+') echo "ticket=$TICKET" >> $GITHUB_OUTPUT
